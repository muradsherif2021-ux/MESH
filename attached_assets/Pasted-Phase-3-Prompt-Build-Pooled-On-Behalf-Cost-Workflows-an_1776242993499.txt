==================================================
2) CORE BUSINESS RULE — MANDATORY
==================================================

The company often pays or records costs in pooled total amounts that serve multiple customers.

Examples:
- one agent trip may serve multiple shipments/customers
- one customs payment may cover multiple customer shipments
- one additional fee batch may relate to several customer jobs
- one field advance may eventually be distributed across more than one customer if business reality requires

Therefore:
- these costs must NOT be assigned to one customer by default
- these costs must first be recorded as pooled recoverable on-behalf costs
- actual customer share will be determined later at invoice time

This business rule is mandatory and must shape the data model and workflows.

5) DESIGN PRINCIPLE — COST SOURCE FIRST
==================================================

The system must use a unified cost source model.

Every recoverable on-behalf cost must generate or map to a cost source record.

Cost source examples:
- AGENT_TRIP
- AGENT_EXTRA_FEE
- CUSTOMS_PAYMENT
- FIELD_ADVANCE
- OTHER_ON_BEHALF_COST

Each cost source represents a recoverable pooled amount that may later be allocated across one or more customer invoices.

This abstraction is mandatory.
Do not scatter pooled balance logic across unrelated modules without a unified source model.

==================================================
10) COST SOURCE ENGINE — MANDATORY
==================================================

Implement a centralized cost source engine/table/service.

Each cost source should include at least:
- id
- sourceType
- sourceId
- sourceNumber
- date
- branchId
- description
- linkedAgentId if applicable
- totalAmount
- allocatedAmount
- remainingAmount
- currency if relevant
- status
- postingState
- createdBy
- postedBy if applicable
- timestamps
- audit linkage

Rules:
- totalAmount >= allocatedAmount
- remainingAmount = totalAmount - allocatedAmount
- remainingAmount can never be negative
- cost source can be:
  - UNALLOCATED
  - PARTIALLY_ALLOCATED
  - FULLY_ALLOCATED
  - CANCELLED if architecture allows
- cost source must be queryable independently of the source document

Important:
This is one of the most important tables/services in the system.
Design it carefully.

- correction should later happen via reversal/correction flow, not unsafe edits
